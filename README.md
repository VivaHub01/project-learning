## Логика диаграммы (Главные сущности)
### В системе проектного обучения можно выделить 6 основных блоков:
* User (Пользователь): Базовый класс для всех участников.
* Project (Проект): Основная единица работы.
* Team (Команда): Группа студентов, работающая над проектом.
* Task (Задача): Конкретные шаги внутри проекта.
* Submission (Сдача/Работа): Результат выполнения задачи (файл, текст).
* Evaluation (Оценка): Результат проверки работы заказчиком/преподавателем.
### Текстовая структура (Главные сущности)

## Описание контейнеров
| Контейнер       | Технологии              | Назначение                                                                                                       |
|-----------------|-------------------------|------------------------------------------------------------------------------------------------------------------|
| Web Application | React, TypeScript       | Клиентское SPA-приложение. Интерфейс для студентов, преподавателей и заказчиков. Общается с API по HTTPS (JSON). |
| Mobile App      | React Native            | Мобильное приложение (опционально). Для уведомлений и быстрого доступа к задачам.                                |
| Backend API     | NocoDB                  | Сердце системы. Обрабатывает запросы, управляет бизнес-логикой, проверяет права доступа (JWT).                   |
| PostgreSQL      | SQL                     | Реляционная база данных. Хранит пользователей, проекты, задачи, оценки, состав команд.                           |
| Valkey          | In-memory store         | Кэширование частых запросов (список проектов), хранение сессий, брокер сообщений для фоновых задач.              |
| File Storage    | Yandex / S3             | Объектное хранилище. Здесь лежат все загруженные файлы: курсовые работы, презентации, аватарки.                  |
| Email Service   | Unisender / SMTP        | Внешний сервис для отправки уведомлений (новая задача, оценка, напоминание).                                     |
| LMS             | Moodle API              | Внешняя система, куда отправляются итоговые оценки в конце семестра.                                             |

## Ключевые особенности системы
* Единая точка входа через ЭИОС ГУУ критически важна для всех пользователей
* Персонализация контента – каждый пользователь видит только релевантные ему проекты
* Система уведомлений – ключевой канал коммуникации о новых проектах, сдачах работ и оценках
* API для экспорта – необходим для интеграции с портфолио студентов и внешними системами
* Ролевая модель – четкое разделение прав доступа обеспечивает безопасность
* Аналитика для руководства – обоснование управленческих решений на основе данных
* Возможность ручной коррекции – важна для администраторов при форс-мажорах
* Прозрачность оценивания – студенты видят критерии и могут отслеживать прогресс
## MVP пользовательского интерфейса (Frontend)
### Экран 1: Авторизация (/login)
* Поля: Email, Пароль
* Кнопка "Вход"
* Ссылка "Регистрация"
* (Упрощение: регистрация только по Email/Паролю, роль "Студент" по умолчанию)
### Экран 2: Список проектов (/projects)
* Карточки проектов с полями:
    - Название проекта
    - Краткое описание
    - Статус (Набор/В работе/Завершен)
    - Преподаватель (автор)
* Кнопка "Создать проект" (только для преподавателей)
* Поиск по названию
### Экран 3: Детали проекта (/projects/:id)
* Полное описание проекта
* Информация о команде (кто уже в проекте)
* Список задач (если есть)
* Кнопки:
    - "Подать заявку" (для студента, если есть место)
    - "Редактировать" (для преподавателя)
    - "Добавить задачу" (для преподавателя)
### Экран 4: Мои задачи (/tasks)
* Список задач, назначенных текущему пользователю
* Для каждой задачи:
    - Название
    - Дедлайн
    - Статус (К выполнению/На проверке/Принято)
    - Ссылка на проект
* Кнопка "Загрузить работу" рядом с задачами в статусе "К выполнению"
### Экран 5: Сдача работы (/tasks/:id/submit)
* Просмотр описания задачи
* Форма загрузки:
    - Комментарий (текстовое поле)
    - Загрузка файла (до 10 МБ)
* Кнопка "Отправить на проверку"
* (Упрощение: файлы хранятся как ссылки в NocoDB, сами файлы — в облаке)
